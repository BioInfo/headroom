# Localization (i18n)

**Status: infrastructure in place, translations are the fast-follow.**

Headroom is set up to be localized. What's done and what's left:

## Done (the capability)

- `Package.swift` declares `defaultLocalization: "en"`.
- A String Catalog ships in the app target: `Sources/HeadroomApp/Resources/Localizable.xcstrings`.
- SwiftUI `Text("…")` literals are already `LocalizedStringKey`s, so the app stays English
  today (keys fall back to their literal) and becomes translatable the moment locale entries
  are added to the catalog.

## Left to do (the fast-follow)

1. **Extract the strings.** `swift build` doesn't run Xcode's catalog extraction. Either open
   the package in Xcode once (Product → Build populates the catalog from `Text` literals), or
   add keys to `Localizable.xcstrings` by hand. Interpolated strings (e.g. "92% used") and
   non-`Text` strings (accessibility labels, `.help(…)`) need `String(localized:)` /
   `LocalizedStringKey` wrapping as they're converted.
2. **Translate the top 10.** Target set (typical macOS dev-tool priority):
   Spanish `es`, Simplified Chinese `zh-Hans`, French `fr`, German `de`, Japanese `ja`,
   Brazilian Portuguese `pt-BR`, Russian `ru`, Korean `ko`, Italian `it`, Hindi `hi`.
   Machine-draft each, then a native spot-check before shipping (UI strings are short and
   context-sensitive — bad translations read worse than English).
3. **Wire it into the app bundle.** SwiftPM puts resources in the module bundle
   (`Bundle.module`), but SwiftUI `Text` localizes against `Bundle.main`. `build-app.sh` must
   copy the compiled `*.lproj` / `Localizable.loctable` into `Headroom.app/Contents/Resources`
   so the main bundle carries them. (Until then, translations live in the module bundle and
   won't be picked up by default `Text`.)

## Why opt for infra-now

"Setup to allow the top 10 languages" is the capability: the package, the catalog, and the
string-as-key flow. Translations + the bundle-copy step are mechanical follow-ups that don't
block anything and are better done in one focused pass with a native check.
